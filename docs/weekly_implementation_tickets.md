# 今週の実装チケット（1チケット=半日〜1日）

## Ticket 1（0.5日）
**目的:** 画面骨格を Today だけでなく History / Settings まで拡張する。  
**実装:** Bottom Navigation で3画面切替を追加。  
**完了条件:** Today/History/Settings が遷移可能。

## Ticket 2（1日）
**目的:** 追加導線を実運用に近づける。  
**実装:** Add Habit ダイアログに「タイトル・通知時刻・Webリンク・アプリリンク」を追加。  
**完了条件:** 入力内容が保存され、Today リストへ反映。

## Ticket 3（0.5日）
**目的:** リンク起動の安全性を上げる。  
**実装:** Web URL とアプリリンク入力のバリデーションを追加。  
**完了条件:** 不正値は保存されず、エラー表示される。

## Ticket 4（0.5日）
**目的:** Repository 経由の保存処理を拡張する。  
**実装:** Habit 作成時に schedule / links も同時登録。  
**完了条件:** habits + habit_schedules + habit_links が整合して作成される。

## Ticket 5（0.5日）
**目的:** データアクセス効率を改善する。  
**実装:** links の全件取得を廃止し、表示対象 habit のみ一括取得する。  
**完了条件:** 1画面のリンク取得クエリが必要対象のみになる。

## Ticket 6（0.5日）
**目的:** リマインダー継続性を強化する。  
**実装:** 新規作成時に reminder 設定があれば再スケジュールを実行。  
**完了条件:** reminder 付き habit 追加後に次回通知が登録される。

## Ticket 7（0.5日）
**目的:** DI/テスト容易性を改善する。  
**実装:** ViewModel に Factory を導入し、依存注入可能な構造へ変更。  
**完了条件:** ViewModel 生成時に Repository/Scheduler を外部から注入可能。

## Ticket 8（0.5日）
**目的:** UX品質を底上げする。  
**実装:** ハードコード文言を resources 化、主要アイコンの contentDescription 調整。  
**完了条件:** 文字列直書きが減り、アクセシビリティ情報が改善される。
