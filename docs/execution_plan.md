# HabHub 実行計画（課題を1つずつ解消）

## フェーズ1: 実行可能な導線を先に作る（UI）
1. Today 画面の `+` を有効化し、ハビット追加ダイアログを実装する。  
   - 完了条件: アプリ上でタイトルを入力して保存するとリストに反映される。
2. リンク起動時の例外処理を実装する。  
   - 完了条件: 起動不能リンクでクラッシュせず、Snackbar でエラー通知する。

## フェーズ2: データ取得を改善する（DAO/Repository）
3. Today 表示用のJOINクエリを導入し、完了状態・通知時刻をまとめて取得する。  
   - 完了条件: Repository でN+1を避けたデータ生成になる。
4. リンク取得を「ハビット単位」ではなく「一覧一括取得」にする。  
   - 完了条件: 1画面描画時のリンク取得クエリが1回に集約される。

## フェーズ3: リマインダーの継続性を上げる
5. ReminderWorker が通知表示後に翌日分を再スケジュールする。  
   - 完了条件: 単発実行で終わらず、毎日継続する。
6. 通知内容に habit タイトルを反映する。  
   - 完了条件: 通知テキストが固定文のみにならない。

## フェーズ4: ドキュメントと整合性
7. README を現実装に合わせ更新する。  
   - 完了条件: 追加可能/リンク失敗時挙動/通知再スケジュールの記述がある。

## フェーズ5: 検証
8. 静的確認とビルド確認を実施。  
   - 完了条件: `git diff --check` は成功。`assembleDebug` は可能な限り確認し、環境制約があれば明記する。
